Agent 기반 통계 자료 수집·검증·시각화 운영 절차

웹에서 지표를 찾고 신뢰성 있게 시각화하기 위해, Agent는 아래 4단계 규칙으로 작업을 표준화합니다.

1) 자료 찾기 알고리듬

  • 질문을 지표/단위/범위/기간/빈도/비교방식으로 분해
  • KOSIS 우선, 국제 비교는 IMF/World Bank 보완
  • 키워드 반복 탐색 후 후보 테이블 ID 소범위 점검
  • 기간·주기·항목의미·코드명확성·결측으로 후보 점수화

2) 자료 검증 알고리듬

  • 스키마(PRD_DE/DT/ITM_ID/C1,C2/PRD_SE/UNIT_NM) 확인
  • 시계열 기간, 중복키, 결측률 검증
  • 코드 의미 교차확인 및 파생지표 계산식 명시
  • 집계 합계 샘플 검증 + 차트 바인딩/범례/footnote 점검
3) 실무 프롬프트 패턴
P1 테이블 찾기: 후보 ID + 기간/주기/단위/ITM_ID 비교
P2 최종 선정: 기간·주기·범위를 가장 충족하는 1개 선정 근거 제시
P3 집계 검증: 코드집합 합산 시계열과 원자료 합계 검증 로그 출력
P4 비교 차트: 연도범위 정렬 + 소스/항목코드 footnote 포함
P5 품질 리포트: 기간·결측·중복·단위·코드 의미 체크리스트 출력

4) 재현 가능한 운영 규칙

  • 한 파일 누적 원칙: 수집 + 검증 + 렌더 통합
  • 테이블 ID/항목코드/지역코드 상수화
  • 생성 직전 period min/max, rows, group count 출력 의무화
  • 모든 차트에 캔버스 내부 footnote로 소스 명시
