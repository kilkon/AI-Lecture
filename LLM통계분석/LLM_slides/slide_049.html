웹 구조를 알아야 API를 찾을 수 있다

탐색 체크리스트

  • 브라우저 개발자도구(Network)에서 XHR/Fetch 요청 확인
  • 응답이 JSON인지, HTML 조각인지 먼저 구분
  • 쿼리 파라미터(기간, 지역, 코드)와 헤더(인증) 기록
  • DOM에 없는 데이터는 JS 호출(API)로 받아오는 경우가 많음

실무 결정 규칙

  • 동일 데이터가 API와 화면 모두에 있으면 API 사용
  • API가 없고 정적 HTML이면 selector 기반 스크래핑
  • 동적 렌더링/클릭 기반이면 Selenium 자동화
  • robots.txt, 이용약관, 호출 빈도 제한을 항상 준수
Example workflow: Open DevTools → Network(XHR) → find endpoint → reproduce in Python requests → validate JSON schema → save raw + cleaned data with timestamp
