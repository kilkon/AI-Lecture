API로 프롬프트 구현하기

프롬프트 구성요소 & ChatGPT API(Chat Completions) 기준 예시

프롬프트 = 역할·과업·입력·제약·출력·기준을 담은 지시문. OpenAI API에서는 system / user(다중 턴 가능)로 나누어 넣는다.

ChatGPT API에서의 대응 관계

Chat Completions API는 대화를 messages 배열로 전송한다.
  • role: "system" — 통상 역할(Role), 지속적 제약(Constraints), 금지사항·톤을 배치하기에 적합하다.
  • role: "user" — 맥락(Context), 과업(Task), 입력자료(Input), 출력형식(Output), 평가기준(Criteria)을 단일 블록 또는 여러 턴으로 전달한다.
  • temperature, max_tokens 등 — 창의성·길이에 대한 통제(control)에 해당한다.
모델명은 예시로 제시한 gpt-4o-mini 등을 실제 사용 중인 엔드포인트에 맞게 변경한다.

1. 프롬프트의 기본 구성요소

1역할 (Role)

AI가 어떤 관점·전문성으로 응답할지 지정한다. 어조, 분석 수준, 설명 방식에 영향을 미친다.

예:
  • “당신은 행정학 연구방법론 전문가이다.”
  • “당신은 통계분석을 도와주는 연구조교이다.”

API 팁: system에 한두 문단으로 고정하면 턴마다 일관성을 유지하기 쉽다.

2과업 (Task)

실제로 수행할 핵심 요구—“무엇을 하라”에 해당한다. 구체적이고 관찰 가능하게 기술하는 것이 바람직하다.

예:
  • “이 연구주제를 바탕으로 연구질문 5개를 제안하라.”
  • “다음 회귀분석 결과를 해석하라.”
  • “선행연구를 비교표 형태로 정리하라.”

3맥락 (Context)

배경, 상황, 대상, 목적 등을 뜻한다. 부재할 경우 응답이 지나치게 일반화될 수 있다.

예:
  • 연구 분야: 지방정부 디지털 전환
  • 대상: 행정학 석사논문
  • 데이터: 설문자료 500명
  • 목적: 연구모형 설계 및 가설 도출

4입력자료 (Input)

분석·변환의 대상이 되는 구체 자료(초록, 표, 회귀 출력, 코드 등)이다.

예:
  • “다음 초록을 요약하라.”
  • “아래 회귀분석 결과를 해석하라.”
  • “다음 코드를 수정하라.”

5출력형식 (Output Format)

표, 번호 목록, JSON, APA, H1/H2 가설 형식 등 출력 구조를 명시한다.

예:
  • 표 형태로 정리 / 번호 목록 / 논문식 문단
  • APA 양식 참고문헌 포함
  • H1, H2 형식의 가설로 작성

6제약조건 (Constraints)

분량, 용어 수준, 금지사항, 분석 범위 등 반드시 준수할 조건이다.

예:
  • “학술논문 문체로 작성하라.”
  • “200자 이내로 요약하라.”
  • “존재하지 않는 문헌은 제시하지 말라.”
  • “행정학 이론 중심으로 설명하라.”

API 팁: system에 “환각 인용 금지” 등을 넣으면 매 요청마다 재사용하기 용이하다.

7평가기준·성공조건 (Criteria)

어떤 응답을 우선할지에 대한 우선순위를 제시한다.

예:
  • “가장 중요한 변수부터 우선 제시하라.”
  • “정확성과 학술적 엄밀성을 우선하라.”
  • “정책적 함의까지 포함하라.”

2. 구조화된 프롬프트의 확장 구성

학술·연구용에서는 아래와 같이 번호를 매겨 한 번에 전달할 수 있으며, API에서는 섹션 제목을 붙인 단일 user 문자열로 전달해도 된다.

  1. 역할: 당신은 행정학 연구방법론 전문가이다.
  2. 맥락: 나는 지방정부 디지털 전환과 주민 신뢰의 관계를 연구하는 석사과정생이다.
  3. 과업: 이 주제에 적합한 연구질문 5개를 제안하라.
  4. 조건: FINER 기준을 반영하고, 행정학 이론과 연결하라.
  5. 출력형식: 각 질문마다 연구의의와 가능한 분석방법을 함께 제시하라.

3. 프롬프트 구성요소의 핵심 원리

  • 명확성(Clarity): 무엇을 요구하는지 분명하여야 한다.
  • 구체성(Specificity): 분야·대상·수준·형식을 구체화한다.
  • 맥락성(Contextualization): 논문용·강의용·정책보고용 등 용도에 따라 기대 응답이 달라져야 한다.
  • 통제가능성(Control): 형식·길이·금지사항으로 산출물을 통제한다.

4. 학술 연구에서 특히 중요한 구성요소

  • 이론적 관점의 명시
  • 자료와 분석단위의 명시
  • 방법론적 제약의 제시
  • 인용 및 출처 검증 요구
  • 학술문체와 논리적 구조 요구

예: 단순히 “이 주제를 설명해줘”에 그치는 것보다,

“행정학 이론을 중심으로, 지방정부 디지털 전환과 주민 신뢰 간 관계를 설명하고, 가능한 매개·조절변수를 포함하여 학술논문식으로 서술하라.”

처럼 역할·맥락·과업·형식을 한 번에 주는 편이 낫다.

5. ChatGPT API 예시 (Python)

OpenAI Python SDK 기준이다. OPENAI_API_KEY 환경변수에 키를 두고 실행한다.

# pip install openai
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# --- 구성요소를 system / user 로 분리한 예시 ---
system_prompt = """당신은 행정학 연구방법론 전문가이다.
응답은 학술 논문 수준의 문체를 사용한다.
존재하지 않는 논문이나 통계를 만들어내지 말고, 불확실하면 명시한다."""

user_prompt = """[맥락] 나는 지방정부 디지털 전환과 주민 신뢰를 연구하는 석사과정생이다.

[과업] 이 주제에 적합한 연구질문(RQ) 후보 5개를 제안하라.

[조건] 각 RQ는 FINER(F,I,N,E,R) 관점에서 한 줄씩 타당성을 붙여라.
행정학 이론(신뢰, 거버넌스, 전자정부 등)과 연결하라.

[출력형식] 각 항목을 번호 목록으로. 각 RQ 다음에 ▪ 연구의의 ▪ 가능한 분석방법 을 bullet로."""

response = client.chat.completions.create(
    model="gpt-4o-mini",  # 또는 gpt-4o 등
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_prompt},
    ],
    temperature=0.4,   # 창의성 vs 일관성 (낮을수록 보수적)
    max_tokens=2000,
)

print(response.choices[0].message.content)

입력자료가 있을 때: 동일 user_prompt 하단에 [입력자료]\n---\n(붙여넣기) 블록을 추가한다.

정리 — 프롬프트 구성요소 7가지
  1. 역할 (Role)
  2. 과업 (Task)
  3. 맥락 (Context)
  4. 입력자료 (Input)
  5. 출력형식 (Output Format)
  6. 제약조건 (Constraints)
  7. 평가기준 (Criteria)

한 줄로: 누가·무엇을·어떤 상황·어떤 자료·조건·형식·기준.
