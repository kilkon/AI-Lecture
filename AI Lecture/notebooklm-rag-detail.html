NotebookLM

기능 소개 및 RAG(검색 증강 생성)의 작동 방식

한 줄 요약: NotebookLM은 업로드한 문헌만을 검색 범위로 쓰는 RAG에 가깝다. 출처 연결이 쉽고, 열린 웹 챗봇보다 허위 인용 위험이 작다.

서비스 열기·화면

개인 Google 계정으로 바로 쓰거나, 모바일 앱·도움말·Workspace 소개 페이지로 이동할 수 있다.

NotebookLM 열기 (웹) Workspace 제품 소개 도움말 (한국어) Google Play App Store

NotebookLM이란?

Google의 문헌 기반 AI 노트. PDF·Docs·URL 등을 한 노트북에 모아 질문·요약·비교·오디오 요약 등을 돌린다.

지원 소스 (업로드 가능 자료)

📄 PDF 논문·보고서
📝 Google Docs
🌐 웹 URL
🎥 YouTube 영상
🎙️ 오디오 파일
📊 텍스트 파일

텍스트 추출·색인이 가능한 형태이면 해당 자료는 노트북의 검색 대상 말뭉치(corpus)에 편입된다.

주요 기능

  • 자동 요약 — 여러 소스를 묶어 개요·쟁점 정리
  • 인용·출처 추적 — 응답과 원문 조각 연결
  • 소스 간 비교 — 논문 A vs B 관점·방법·결과
  • 오디오 오버뷰 — 업로드 내용 팟캐스트형 음성 요약
  • 심층 연구(Deep Research) — (제공 시) 소스 기반 심화 탐색
  • Few-shot 문헌 분류 — 예시를 제시한 뒤 동일 형식으로 표·분류를 요구하는 반복 작업

RAG는 NotebookLM에서 어떻게 이루어지나?

RAG란?

RAG(Retrieval-Augmented Generation): 문서 조각을 검색해 프롬프트에 붙인 뒤 생성하는 방식이다.

NotebookLM에 적용할 때의 흐름 (개념 모델)

내부 구현은 비공개이나, 통상적인 RAG 단계로 설명한다.

① 인덱싱 단계 (소스가 노트북에 들어올 때)
파일·URL에서 텍스트 추출 → 장문은 청킹 → 조각별 임베딩 → 벡터 인덱스에 저장.
📚 사용자 소스
PDF·Docs·URL… → 텍스트 추출
& 청킹 → 임베딩 &
벡터 인덱스
▼ 질문할 때 (런타임) ▼
💬 사용자 질문 → 질문 임베딩
유사도 검색 → 상위 k개
문단 회수 → LLM에
「근거 문단」첨부 → 답변 +
출처 표시
② 검색·생성 (질문 시)
질의를 벡터화해 유사 청크를 고르고, 그 텍스트만 LLM에 넣어 답을 만든다. 근거는 업로드된 문헌 안에 묶인다.

일반 ChatGPT vs NotebookLM (RAG 관점)

일반 챗봇(웹 검색 없을 때)
검색 단계 없이 파라미터만으로 답하면 허위 인용·통계가 나오기 쉽다.
NotebookLM
검색 범위가 업로드 소스로 한정된다. 출처 링크로 원문을 열어 볼 수 있다.

한계 (연구자가 알아둘 점)

  • 검색이 빗나가면 엉뚱한 문단을 근거로 쓰거나, 없는 내용을 보강할 수 있다 → 원문 대조는 필수.
  • 청크 경계 때문에 문맥이 잘린 문단만 들어가 오해가 날 수 있다.
  • 올리지 않은 자료는 답에 나오지 않는다.
