프롬프트 형성기(Prompt Builder) 설계 고려사항

대학원생 대상 · 지식 생산을 돕는 지적 도구로 설계하기

핵심: 단순 문자열 조합기가 아니라, 연구·학습 맥락에서 재사용 가능한 지시문을 체계적으로 만드는 도구로 본다. 아래는 기획·설계 시 참고할 수 있는 정리이다.

I. 설계 철학 — 무엇을 만들 것인가

형성기의 방향을 먼저 정한다. 대략 세 유형으로 나눌 수 있다.

유형 개념 적합 대상
① 템플릿형 미리 만든 틀의 빈칸을 채움 입문, 빠른 사용
② 가이드형 단계별 질문에 답하면 프롬프트 완성 중급, 학습 병행
③ 프레임워크형 이론 기반 구조(예: PACE, RCTF 등)로 조립 연구자·고급 사용자
대학원생 대상 권장: ②+③ 혼합 — 가이드 질문으로 입력을 받고, 선택한 프레임워크에 맞춰 조립하는 방식.

II. 핵심 구성 요소 — 최대 8요소

형성기가 어떤 칸을 노출할지 결정할 때 쓸 수 있는 분해표이다. ROLE·TASK·FORMAT을 필수에 두고, 나머지는 선택 옵션으로 두는 식이 일반적이다.

요소 의미 예시
ROLE AI 역할 「행정학 교수로서」
CONTEXT 배경·맥락 「한국 행정학 석사 과정」
TASK 구체 과제 「연구 질문 5개 생성」
FORMAT 출력 형식·구조 「표 형식으로」
TONE 어조·스타일 「학술적으로」
LENGTH 분량·깊이 「각 항목 3줄 이내」
EXAMPLE 참고 예시 「다음 형식 참고: …」
CONSTRAIN 제약·금지 「추측 금지, 출처만」

필수 최소: ROLE + TASK + FORMAT · 나머지 5개는 과제에 따라 추가.

III. 프레임워크 선택

대학원생용 형성기에 얹을 수 있는 대표 패턴 비교.

프레임워크 구성 강점 적합 용도
P.A.C.E. Purpose · Action · Context · Explain 직관적, 빠른 학습 일반 학습 프롬프트
R.C.T.F. Role · Context · Task · Format 역할·구조가 분명 연구·분석 프롬프트
T.A.G. Task · Action · Goal 목표 지향, 단순 단일 과제
C.R.E.A.T.E. Context · Role · Explain · Action · Type · Examples 구성이 촘촘함 복합 연구 작업
S.P.I.N. Situation · Problem · Implication · Need 문제 해결 지향 정책·행정 분석
연구용 조합 예: R.C.T.F.를 기본 골격으로 두고, EXAMPLE + CONSTRAIN을 심화 옵션으로 연결.

IV. 사용 목적별 분류(6카테고리)

형성기에서 “목적 선택” 단계에 쓸 수 있는 상위 분류 예시이다.

[1] 학습·이해

  • 개념 설명, 비교 분석
  • 시험 준비, 오답 분석

[2] 연구·논문

  • 연구 질문·문헌 갭·가설
  • 방법론 선택, 논문 피드백

[3] 과제·발표

  • 보고서 개요, 논증 구성
  • 발표 스크립트, Q&A 준비

[4] 커리어·개발

  • 역량 진단, 학습 로드맵
  • 자기소개서, 면접 준비

[5] 정책·행정

  • 정책 분석, 거버넌스·규제
  • 사례 분석

[6] 비판·검증

  • AI 출력 사실 확인, 논리 오류
  • 편향·윤리 점검

V. UX 흐름(4단계)

  1. 목적 선택 — 위 카테고리 카드(또는 유사 UI)에서 선택. 「무엇을 하려는가?」를 고정.
  2. 세부 입력 — 가이드형 질문: 분야/주제, AI 역할, 출력 형식, 분량·수준, 제약(선택) 등.
  3. 생성·미리보기 — 조합된 프롬프트 표시, 복사·수정·저장 제공.
  4. 자가 점검 — 역할·맥락·과제·형식·윤리 등 체크리스트(아래 VIII과 연계 가능).

생성 결과 예시 블록은 연구 주제에 맞게 변수 치환한 문자열로 보여 주면 된다.

당신은 {role}입니다. {context} 맥락에서 {task}을(를) {format}에 맞게 작성하세요. {constraints}

VI. 기술 구현 옵션

방식 도구·기술 난이도 특징
노코드형 Notion, Google Forms + Apps Script 등 낮음 빠른 프로토타입, UX 제약
로우코드형 Bubble, Glide, Webflow 등 중간 중간 수준 커스터마이징
풀코드형 HTML·CSS·JS 정적 페이지 중~높음 제어·오프라인 배포 용이
AI 연동형 OpenAI 등 API + 웹앱 높음 생성·품질 피드백 자동화 가능

데이터 구조 예시(JSON)

{
  "prompt_builder": {
    "category": "research",
    "subcategory": "research_question",
    "components": {
      "role": "행정학 전문 교수",
      "context": "한국 전자정부 연구, 석사 과정",
      "task": "연구 가능한 질문 5개 생성",
      "format": "표 형식 (질문/방법론/난이도)",
      "length": "각 항목 2줄 이내",
      "constraints": "실증 연구 가능한 주제만"
    },
    "output_template": "당신은 {role}입니다. {context} 상황에서 {task}을 {format}으로 작성해주세요. {constraints}",
    "quality_score": null,
    "ethics_check": true
  }
}

VII. 윤리·책임(형성기에 넣을 장치)

  1. 사용 목적 선언 — 「이 프롬프트는 [ ] 목적으로 쓴다」(학습 보조 / 연구 지원 / 과제 제출 등). 과제 전체 대필에 가까우면 별도 경고.
  2. AI 사용 표기 리마인더 — 제출물에 AI 사용 사실을 표기하도록 안내.
  3. 금지·고위험 용도 경고 — 학문적 진실성에 어긋날 수 있는 패턴에 ⚠️ 메시지.
  4. 출처 검증 안내 — 출력 오류 가능성, 원문·데이터 직접 확인 문구.
  5. 지식 주체성 — 「본인의 분석·판단이 포함되었는가?」 확인 질문.

강의의 SPARRO·투명성 보고(6·28쪽)와 맞춰 두면 좋다.

IX. 기획 전 확인할 5가지 질문

  • Q1. 누가 쓰는가? → 타겟 페르소나
  • Q2. 어디서 쓰는가? → 웹·오프라인 HTML·노션 등
  • Q3. 몇 개 카테고리를 지원할 것인가? → 초기에는 소수 권장
  • Q4. AI API를 붙일 것인가? → 실시간 피드백·생성 자동화 여부
  • Q5. 윤리 경고를 어떻게 넣을 것인가? → 대학원생 대상이면 필수에 가깝다
