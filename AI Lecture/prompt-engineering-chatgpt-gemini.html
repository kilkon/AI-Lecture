🤖 프롬프트 엔지니어링: ChatGPT vs Gemini 핵심 요소

CO-STAR·PARTS·Gemini 공식 전략·공통 기법 요약

같은 모델이라도 프롬프트에 따라 출력 품질 차이가 크다. 아래는 정리용 체크리스트다.

용어 정리: 「PARTS」는 두 가지 뜻으로 쓰인다. (1) 교육·콘텐츠 분야의 5단계 약어 프레임워크 PARTS(Persona–Act–Recipient–Theme–Structure), (2) 프롬프트를 지칭할 때의 구성 요소(parts of a prompt)—명령·맥락·페르소나 등 6가지 핵심 요소. 아래에서 양쪽을 정리한다.

🟢 ChatGPT — CO-STAR 프레임워크

CO-STAR 여섯 칸 채우기. (달파)

요소 영문 설명 예시
🔵 맥락 Context 요청의 배경·상황 「나는 스타트업 마케터이고, 신제품 출시를 앞두고 있다」
🎯 목적 Objective 달성하려는 목표 「인스타그램용 홍보 문구 3개를 만들어줘」
🎨 스타일 Style 답변 형식·문체 「블로그 형식, 친근하고 감각적인 문체」
🎭 어조 Tone 감성·분위기 「열정적이고 긍정적인 어조」
👥 독자 Audience 결과를 읽을 대상 「20~30대 MZ세대 여성」
📄 응답형식 Response 출력물의 구체적 형태 「각 문구는 2줄 이내, 해시태그 5개 포함」

➕ ChatGPT 추가 설정 요소

  • 🛠 Custom Instruction (맞춤 설정) — 직업, 코딩 스타일, 선호 형식 등을 미리 저장해 매번 반복 입력을 줄임 (웹 UI·앱).
  • 📝 마크다운 포맷 활용 — #, -, ** 등으로 프롬프트 구조를 나누면 문맥 파악에 유리함.
  • 🌡 Temperature (온도) — 창의 작업 0.7~1.0, 사실 전달 0.3~0.5, 코드 생성 0.2~0.4 권장(일반적 가이드; 과제에 맞게 조정).
  • 🎭 Role (역할) — 「당신은 10년 경력의 카피라이터이다」와 같이 구체적 역할을 시스템 메시지 또는 맞춤 설정에 두면 일관성이 높아진다.

📦 PARTS 프레임워크 (5단계 약어)

PARTS 5단계 약어(Persona–Act–Recipient–Theme–Structure). 아래 「6대 구성요소」와 이름만 비슷하고 뜻은 다름.

글자 영문 한국어 핵심 질문
P Persona 페르소나 (역할) AI가 누구로서 답해야 하는가?
A Act 액션 (행동) AI가 무엇을 해야 하는가?
R Recipient 수신자 (대상) 결과물을 받을 사람은 누구인가?
T Theme 주제 (테마) 어떤 주제·내용을 다루는가?
S Structure 구조 (형식) 어떤 형태로 출력해야 하는가?

✏️ PARTS 실전 예시

[P - Persona] 당신은 10년 경력의 시니어 마케터이다. [A - Act] 다음 제품의 인스타그램 홍보 게시물을 작성해줘. [R - Recipient] 20~30대 직장인 여성을 대상으로 해줘. [T - Theme] 신제품 에너지 드링크 '모닝부스트' 출시 캠페인. [S - Structure] • 캡션 3문장 이내 • 이모지 3개 포함 • 해시태그 5개 포함 • 행동 유도 문구(CTA) 1개 포함

프롬프트의 핵심 구성 요소 (Parts of a Prompt)

「프롬프트를 구성하는 parts」를 말할 때는 아래 6가지를 자주 구분한다. 참고: AI Ground · Learn Prompting — Prompt structure

# 요소 설명 예시
① 명령 Task AI에게 정확히 무엇을 해야 할지 지시 (필수) 「~작성해줘」「~요약해줘」「~번역해줘」
② 맥락 Context 배경 상황, 조건, 규칙 「이 글은 사내 발표용이야, 전문 용어 사용 OK」
③ 페르소나 Persona 역할·전문가 설정 「회계사 입장에서 답변해줘」
④ 예시 Example 기대 출력의 샘플 (Few-shot) 「예시: 제목 - 본문 - 결론 형태처럼」
⑤ 포맷 Format 출력 형태·분량·구조 「마크다운 표로 정리」「500자 이내」
⑥ 어조 Tone 말투·분위기·스타일 「친근한 말투로」「전문적이고 간결하게」

PARTS vs 다른 프레임워크 비교

프레임워크 구성 특징
PARTS Persona · Act · Recipient · Theme · Structure 교육·콘텐츠 제작에 맞춤
CO-STAR Context · Objective · Style · Tone · Audience · Response ChatGPT 활용에 널리 알려짐, 요소가 종합적
CARE Context · Action · Result · Example 비즈니스 업무용으로 소개되는 사례가 많다
RTF Role · Task · Format 가장 간결, 빠른 반복 작업에 적합
PACT Persona · Action · Context · Target 마케팅 카피 등에 인용되는 패턴

💡 PARTS·구성요소 활용 팁

모든 요소를 매회 전부 채울 필요는 없다.
• 빠른 작업: P + A + S (페르소나·행동·구조)만으로도 충분한 경우가 많다
• 고품질 콘텐츠: P + A + R + T + S (PARTS 5가지)를 채운다
• 코드 작업: A + T + S (무엇을 할지·주제·출력 구조) 중심으로 시작한 뒤 필요 시 맥락·예시를 추가한다

연구용 기본 순서: ①명령 + ②맥락 + ⑤포맷, 필요 시 ③페르소나·④예시. (API 7요소)

🔵 Gemini — Google 공식 설계 전략

Google AI for Developers의 프롬프팅 가이드에 맞춘 요지이다. (Google AI — Prompting strategies (한국어))

1️⃣ 명확하고 구체적인 요청
모호한 지시 ❌ → 단계별·구체적 지시 ✅
「뭔가 재밌는 거 써줘」 ❌ → 「20~30대를 위한 자기계발 관련 유머 3개를 작성해줘」 ✅
2️⃣ 제약조건(Constraint) 명시
  • 응답 길이 제한 (예: 「500자 이내」)
  • 해야 할 것 / 하지 말아야 할 것 구분
  • 특정 단어·형식 사용 여부 제한
3️⃣ 응답 형식(Output Format) 지정
JSON, 표, bullet, 번호 목록 등 형식을 밝힌다.
4️⃣ 퓨샷(Few-shot) 예시
좋은 예시 몇 개를 넣는다. 나쁜 예보다 따를 본보기를 보여 주는 쪽이 낫다.
5️⃣ 컨텍스트(Context) 정보 추가
  • 배경 지식, 도메인 정보
  • 관련 자료·데이터 첨부(멀티모달)
  • 이전 대화 맥락 유지(멀티턴)
6️⃣ 프리픽스(Prefix) 설정
입력:, 출력:, 요약:, 번역: 등 레이블로 입력·출력 구역을 나누면 모델이 맥락을 파악하기 쉽다.
7️⃣ 모델 파라미터 조정 — 아래 표 참고 (API·Vertex 등).

Gemini — 모델 파라미터 (요약)

파라미터 역할 권장·참고
Temperature 창의성·무작위성 시작값 0.2 부근에서 시도 후 과제에 맞게 조정
Top-K 후보 토큰 수 제한 낮을수록 보수적·예측 가능한 응답에 가까움
Top-P 누적 확률 기반 샘플링 0.8~0.95 구간이 자주 쓰임(모델별 문서 확인)
Max Output Tokens 최대 출력 길이 작업 규모·비용에 맞게 설정

⚡ 공통 고급 기법 (두 모델 모두 적용)

🧠 Chain-of-Thought (CoT) → 단계를 쓰게 한 뒤 답하게 한다(추론·산술에 자주 씀). 🔗 프롬프트 체이닝 → 복잡한 작업을 여러 단계로 나눠 순차 실행 (요약 → 분류 → 작성 등) 🔁 반복 최적화 (PDCA) → 프롬프트 → 실행 → 결과 분석 → 수정을 3회 이상 반복 🎭 역할(Persona) 부여 → 전문가 페르소나로 깊이·용어 수준 조절 📊 구조화 출력 강제 → JSON·표 형식으로 일관된 결과·후처리 용이

파라미터 접근 Playground·API·일부 클라이언트에서 조정 API·Vertex AI 등에서 세밀 제어
사전 설정 기능 Custom Instructions System Instruction

    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "당신은 행정학 연구방법론 조교이다."},

    ],

)

    "topP": 0.95,
    "topK": 40
  }
}
